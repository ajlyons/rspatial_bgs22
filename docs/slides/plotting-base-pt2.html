Enhancing Basic Plots

Recall our basic plot of Yosemite’s historical places from an earlier section.

## Basic plot of Yosemite historic places
plot(yose_bnd_ll %>% st_geometry(), asp=1)
plot(yose_hp %>% st_geometry(), add=TRUE)

The basic plot() function has dozens of optional arguments you can use to customize a plot. Examples:

  • type - type of plot (p=point, l=line, b=both)
  • pch - plot character (points layers)
  • cex - character expansion factor (1=normal size)
  • lwd - line width
  • lty - line type
  • col - fill
  • border - outline color
  • main - title
  • xlab, ylab - set labels for the x and y axes
  • asp - aspect ratio (set asp=1 for geographic data)
  • axes - whether or not to display axes
  • add - set add=TRUE to overlay new feaures on current plot
  1. Try adding the following arguments to your plot command to see what they do.

axes=T
asp=1
col="cornsilk"
border="chocolate4"
lwd=2
main="Historic Places in Yosemite National Park"
pch=17

[Solution]

plot(yose_bnd_ll %>% st_geometry(), asp=1, col="cornsilk", border="chocolate4", lwd=2, axes=T, main="Historic Places in Yosemite National Park")
plot(yose_hp %>% st_geometry(), pch=17, cex=1.2, col="darkolivegreen", add=TRUE)

We illustrate this by plotting the county boundaries.

This is why we’ve been using the st_geometry() function, which essentially drops the attribute table.

To specify which column to use for symbology, enter the name of the column in square brackets. plot() will select a default symbology based on the data type (e.g., continous vs. discrete).

plot(ca_counties_ll["AWATER"])

What happens when you enclode the name of an attribute column in double square brackets? Can you explain why?

[Answer]

plot(ca_counties_ll[["AWATER"]])

It generates a scatter plot, because double-square brackets returns the column values as a vector of numbers rather than a sf object.

Customizing Symbology

A Choropleth map is a thematic map in which areas are shaded or patterned in proportion to a variable.

plot() will guess how to symbolize the layers based on the data type of the attribute column (i.e., numeric vs character). You can override these options with additional arguments.

Change the number of breaks with nbreaks

Change how breaks are defined (e.g., equal area, equal interval, manual) with breaks

Change the color palette with pal

## Plot the area of water variable with 20 equal interval breaks and a reverse topo color ramp
plot(ca_counties_ll["AWATER"], axes=T, nbreaks=20, breaks="equal", pal=rev(topo.colors(20)))

Additional plot() arguments for sf objects

Change where the legend (aka key) appears by manually setting the key.pos argument to 1 (below), 2 (left), 3 (above), or 4 (right).

Add graticules with graticule=TRUE

Move the legend to the left side and add graticules.

[Solution]

## Make the county map prettier
plot(ca_counties_ll["AWATER"], 
     axes=T, 
     nbreaks=20, 
     breaks="equal", 
     pal=rev(topo.colors(20)), 
     key.pos=2, 
     graticule=TRUE)

If you don’t like RStudio’s tiny plot window, you can open a new plot window by running dev.new(). This works particularly well when you have two monitors.

  • leaflet
  • mapview
  • mapsf
  • ggplot2
  • cartography
  • ggmap
